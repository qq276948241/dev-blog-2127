## 三、状态管理设计（useBlogStore）

全局唯一 Store：[useBlogStore.ts](file:///d:/code/ai-prompt/solo-20/repos/repo41/project41/src/store/useBlogStore.ts)

**为什么选 Zustand**：相比 Redux/Context，Zustand 无需 Provider 包裹、API 极简、天然支持 selector 避免过度重渲染。

### 3.1 Store 里存了什么

```typescript
interface BlogState {
  // ========== 核心数据（来自 Mock） ==========
  articles: Article[];       // 12 篇文章
  categories: Category[];    // 6 个分类
  tags: Tag[];               // 25 个标签
  profile: Profile;          // 个人简历

  // ========== 筛选状态 ==========
  filter: BlogFilter;
  // ├── category: string | null   当前选中的分类 slug
  // ├── tags: string[]            当前选中的标签 slug 数组
  // ├── sort: 'latest' | 'popular' 排序方式
  // └── keyword: string           搜索关键词（搜索页用）

  // ========== 筛选操作 ==========
  setCategory(slug)          // 设置分类（传 null 取消）
  toggleTag(slug)            // 切换标签（点一次选中，再点取消）
  setSort(sort)              // 切换排序
  setKeyword(keyword)        // 设置搜索关键词
  resetFilter()              // 重置所有筛选

  // ========== 派生查询 ==========
  getFilteredArticles()      // 按当前 filter 计算过滤后的文章数组
  getArticleById(id)
  getArticleBySlug(slug)
  getPopularArticles(limit=5)   // 按阅读量取 Top N
  getRelatedArticles(id, limit=3) // 相关推荐（分类2分+标签1分）

  // ========== 副作用 ==========
  incrementViews(id)         // 阅读量 +1（30 分钟内重复访问不计数）
}
```

### 3.2 阅读量防重机制

纯前端场景下，用户反复刷新会把阅读量刷爆。为此实现了 localStorage 冷却：

```
localStorage 键名：blog_visited_articles
存储结构：        { [articleId]: 访问时间戳 }
冷却窗口：        30 分钟（VIEW_COOLDOWN_MS = 30*60*1000）
```

执行流程：
1. `incrementViews(id)` 被调用
2. 调 `shouldCountView(id)` 检查：
   - 从 localStorage 读历史访问时间
   - 如果该 ID 在 30 分钟内访问过 → 返回 `false`，**不计数**
   - 否则写入当前时间戳，并清理所有过期记录 → 返回 `true`，**计数 +1**

### 6.2 完整数据流（用户点击「前端开发」分类为例）

```
用户点击
  │
  ▼
FilterBar 按钮 onClick → setCategory('frontend')
  │                        └─ 来自 useFilteredArticles 的返回
  │
  ▼
useBlogStore.setCategory → set({ filter: { ...filter, category: 'frontend' } })
  │
  ▼ （Zustand 触发订阅者重渲染）
  │
useFilteredArticles 内部 useMemo 依赖变化 → 重新计算：
  1. 调 getFilteredArticles()
       ├─ if (filter.category) → 过滤 a.category === 'frontend'
       ├─ if (filter.tags.length) → 过滤 tags 交集
       ├─ if (filter.keyword) → 匹配 title/summary/tags
       └─ sort: latest(按日期 desc) / popular(按 views desc)
  2. 计算 activeCategory = categories.find(c => c.slug === 'frontend')
  3. 计算 activeTagList / hasFilter
  │
  ▼
HomePage 收到新的 filtered 数组
  │
  ├─ filtered.length 变化 → useEffect 跳回第 1 页
  ├─ 按 PAGE_SIZE(6) slice → pageArticles
  └─ 渲染 ArticleCard × N + Pagination
  │
  ▼
FilterBar 收到新的 activeCategory → 高亮对应分类按钮
       并显示 "当前筛选：分类: 前端开发 ×" 标签条
```

### 6.3 分页联动

- URL 参数同步：`?page=2`
- 筛选条件变化时（`filtered.length` 改变）：自动重置到第 1 页
- 点击分页按钮：`setSearchParams({ page })` + `window.scrollTo({ top: 0 })`

## 七、搜索逻辑流程（搜索页）

搜索页**不使用 store.filter 的 keyword 做文章过滤**，而是在组件内独立计算（避免和首页筛选互相干扰）。

### 7.1 搜索执行流程

```
用户在输入框输入 "Node.js" → 回车提交
  │
  ▼
handleSearch → setSearchParams({ q: 'Node.js' })
  │
  ▼
SearchPage 组件重渲染 → q = searchParams.get('q')
  │
  ├─ 调 useBlogStore.getState().setKeyword(q)  → 仅同步到 store（便于其他地方读取）
  │
  └─ useMemo 计算 results（不依赖 store.getFilteredArticles）：
       articles.filter(a =>
         a.title   .includes(q) ||
         a.summary .includes(q) ||
         a.tags.some(t => t.includes(q)) ||
         a.category.includes(q)
       )
  │
  ▼
渲染：
  - 匹配分类（matchedCategories）：分类名包含 q 的分类
  - 匹配标签（matchedTags）：标签名包含 q 的标签
  - 文章结果（results）：每篇文章的标题/摘要/标签 → 调用 highlight() 做关键词高亮
```

### 7.2 关键词高亮 highlight()

**特殊字符转义**：正则构造前必须转义 `. * + ? ^ $ { } ( ) | [ ] \` 这些元字符，否则用户搜索 `(` 或 `.` 会报错。

```typescript
// 修复前（有隐患）：
const regex = new RegExp(`(${kw})`, 'gi');
parts.map(p => regex.test(p) ? <mark>p</mark> : p)
// 问题：带 g 的 regex.test() 会递增 lastIndex，多次调用漏判

// 修复后：
const escaped = kw.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
const splitRegex = new RegExp(`(${escaped})`, 'gi');   // 只用于 split
const matchRegex = new RegExp(escaped, 'i');          // 不带 g，test 无状态
parts.map(p => p && matchRegex.test(p) ? <mark>p</mark> : p)
```

## 八、文章详情页相关推荐算法

[getRelatedArticles()](file:///d:/code/ai-prompt/solo-20/repos/repo41/project41/src/store/useBlogStore.ts#L133-L150)

打分规则（简单但实用）：

```
score = 同分类 ? 2 : 0
      + 相同标签的数量 × 1

例：当前文章是「前端 + React + 性能优化」
    A 文章是「前端 + React + TypeScript」→ 分类2分 + React1分 = 3分
    B 文章是「后端 + 性能优化」         → 标签1分 = 1分
    C 文章是「前端」                  → 分类2分 = 2分
排序：A > C > B，取 Top 3
```

## 九、常见开发指南

### 新增一篇文章

1. 选好分类（e.g. `frontend`）→ 编辑对应分类文件 `src/data/articles/frontend.ts`，push 一个 Article 对象
2. 若涉及新标签：在 `tags.ts` 添加，并在对应分类的文章里引用 tag **名称**
3. 若分类计数写死了，更新 `categories.ts` 的 `count` 字段

### 新增一个路由页面

1. 在 `src/pages/` 下建页面组件
2. 在 `App.tsx` 的 `<Routes>` 里加 `<Route path="/xxx" element={<XxxPage />} />`
3. 如果需要在顶部导航显示，去 `Header.tsx` 的 links 数组加一项

### 自定义 Tailwind 工具类

- 主题色/字体/动画 → `tailwind.config.js` theme.extend
- CSS 复杂伪类/玻璃拟态样式 → `src/index.css`（如 `.glass-card`、`.chip`、`.btn-primary`）
